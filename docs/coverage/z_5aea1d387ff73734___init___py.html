54class SocketIO(object): 

55 """Create a Flask-SocketIO server. 

56 

57 :param app: The flask application instance. If the application instance 

58 isn't known at the time this class is instantiated, then call 

59 ``socketio.init_app(app)`` once the application instance is 

60 available. 

61 :param manage_session: If set to ``True``, this extension manages the user 

62 session for Socket.IO events. If set to ``False``, 

63 Flask's own session management is used. When using 

64 Flask's cookie based sessions it is recommended that 

65 you leave this set to the default of ``True``. When 

66 using server-side sessions, a ``False`` setting 

67 enables sharing the user session between HTTP routes 

68 and Socket.IO events. 

69 :param message_queue: A connection URL for a message queue service the 

70 server can use for multi-process communication. A 

71 message queue is not required when using a single 

72 server process. 

73 :param channel: The channel name, when using a message queue. If a channel 

74 isn't specified, a default channel will be used. If 

75 multiple clusters of SocketIO processes need to use the 

76 same message queue without interfering with each other, 

77 then each cluster should use a different channel. 

78 :param path: The path where the Socket.IO server is exposed. Defaults to 

79 ``'socket.io'``. Leave this as is unless you know what you are 

80 doing. 

81 :param resource: Alias to ``path``. 

82 :param kwargs: Socket.IO and Engine.IO server options. 

83 

84 The Socket.IO server options are detailed below: 

85 

86 :param client_manager: The client manager instance that will manage the 

87 client list. When this is omitted, the client list 

88 is stored in an in-memory structure, so the use of 

89 multiple connected servers is not possible. In most 

90 cases, this argument does not need to be set 

91 explicitly. 

92 :param logger: To enable logging set to ``True`` or pass a logger object to 

93 use. To disable logging set to ``False``. The default is 

94 ``False``. Note that fatal errors will be logged even when 

95 ``logger`` is ``False``. 

96 :param json: An alternative json module to use for encoding and decoding 

97 packets. Custom json modules must have ``dumps`` and ``loads`` 

98 functions that are compatible with the standard library 

99 versions. To use the same json encoder and decoder as a Flask 

100 application, use ``flask.json``. 

101 :param async_handlers: If set to ``True``, event handlers for a client are 

102 executed in separate threads. To run handlers for a 

103 client synchronously, set to ``False``. The default 

104 is ``True``. 

105 :param always_connect: When set to ``False``, new connections are 

106 provisory until the connect handler returns 

107 something other than ``False``, at which point they 

108 are accepted. When set to ``True``, connections are 

109 immediately accepted, and then if the connect 

110 handler returns ``False`` a disconnect is issued. 

111 Set to ``True`` if you need to emit events from the 

112 connect handler and your client is confused when it 

113 receives events before the connection acceptance. 

114 In any other case use the default of ``False``. 

115 

116 The Engine.IO server configuration supports the following settings: 

117 

118 :param async_mode: The asynchronous model to use. See the Deployment 

119 section in the documentation for a description of the 

120 available options. Valid async modes are ``threading``, 

121 ``eventlet``, ``gevent`` and ``gevent_uwsgi``. If this 

122 argument is not given, ``eventlet`` is tried first, then 

123 ``gevent_uwsgi``, then ``gevent``, and finally 

124 ``threading``. The first async mode that has all its 

125 dependencies installed is then one that is chosen. 

126 :param ping_interval: The interval in seconds at which the server pings 

127 the client. The default is 25 seconds. For advanced 

128 control, a two element tuple can be given, where 

129 the first number is the ping interval and the second 

130 is a grace period added by the server. 

131 :param ping_timeout: The time in seconds that the client waits for the 

132 server to respond before disconnecting. The default 

133 is 5 seconds. 

134 :param max_http_buffer_size: The maximum size of a message when using the 

135 polling transport. The default is 1,000,000 

136 bytes. 

137 :param allow_upgrades: Whether to allow transport upgrades or not. The 

138 default is ``True``. 

139 :param http_compression: Whether to compress packages when using the 

140 polling transport. The default is ``True``. 

141 :param compression_threshold: Only compress messages when their byte size 

142 is greater than this value. The default is 

143 1024 bytes. 

144 :param cookie: If set to a string, it is the name of the HTTP cookie the 

145 server sends back to the client containing the client 

146 session id. If set to a dictionary, the ``'name'`` key 

147 contains the cookie name and other keys define cookie 

148 attributes, where the value of each attribute can be a 

149 string, a callable with no arguments, or a boolean. If set 

150 to ``None`` (the default), a cookie is not sent to the 

151 client. 

152 :param cors_allowed_origins: Origin or list of origins that are allowed to 

153 connect to this server. Only the same origin 

154 is allowed by default. Set this argument to 

155 ``'*'`` to allow all origins, or to ``[]`` to 

156 disable CORS handling. 

157 :param cors_credentials: Whether credentials (cookies, authentication) are 

158 allowed in requests to this server. The default is 

159 ``True``. 

160 :param monitor_clients: If set to ``True``, a background task will ensure 

161 inactive clients are closed. Set to ``False`` to 

162 disable the monitoring task (not recommended). The 

163 default is ``True``. 

164 :param engineio_logger: To enable Engine.IO logging set to ``True`` or pass 

165 a logger object to use. To disable logging set to 

166 ``False``. The default is ``False``. Note that 

167 fatal errors are logged even when 

168 ``engineio_logger`` is ``False``. 

169 """

202 channel = self.server_options.pop('channel', 'flask-socketio')

258 def on(self, message, namespace=None): 

259 """Decorator to register a SocketIO event handler. 

260 

261 This decorator must be applied to SocketIO event handlers. Example:: 

262 

263 @socketio.on('my event', namespace='/chat') 

264 def handle_my_custom_event(json): 

265 print('received json: ' + str(json)) 

266 

267 :param message: The name of the event. This is normally a user defined 

268 string, but a few event names are already defined. Use 

269 ``'message'`` to define a handler that takes a string 

270 payload, ``'json'`` to define a handler that takes a 

271 JSON blob payload, ``'connect'`` or ``'disconnect'`` 

272 to create handlers for connection and disconnection 

273 events. 

274 :param namespace: The namespace on which the handler is to be 

275 registered. Defaults to the global namespace. 

276 """ 

277 namespace = namespace or '/' 

278 

279 def decorator(handler): 

280 @wraps(handler) 

281 def _handler(sid, *args): 

282 return self._handle_event(handler, message, namespace, sid, 

283 *args) 

284 

285 if self.server: 

286 self.server.on(message, _handler, namespace=namespace) 

287 else: 

288 self.handlers.append((message, _handler, namespace)) 

289 return handler 

290 return decorator

292 def on_error(self, namespace=None):

294

297 event handler raises an exception. The handler function must accept one 

298 argument, which is the exception raised. Example:: 

299 

300 @socketio.on_error(namespace='/chat') 

301 def chat_error_handler(e):

303 

304 :param namespace: The namespace for which to register the error 

305 handler. Defaults to the global namespace. 

306 """ 

307 namespace = namespace or '/'

310 if not callable(exception_handler): 

311 raise ValueError('exception_handler must be callable') 

312 self.exception_handlers[namespace] = exception_handler 

313 return exception_handler 

314 return decorator

357 def event(self, *args, **kwargs): 

358 """Decorator to register an event handler. 

359 

360 This is a simplified version of the ``on()`` method that takes the 

361 event name from the decorated function. 

362 

363 Example usage:: 

364 

365 @socketio.event 

366 def my_event(data): 

367 print('Received data: ', data) 

368 

369 The above example is equivalent to:: 

370 

371 @socketio.on('my_event') 

372 def my_event(data): 

373 print('Received data: ', data) 

374 

375 A custom namespace can be given as an argument to the decorator:: 

376 

377 @socketio.event(namespace='/test') 

378 def my_event(data): 

379 print('Received data: ', data) 

380 """ 

381 if len(args) == 1 and len(kwargs) == 0 and callable(args[0]): 

382 # the decorator was invoked without arguments 

383 # args[0] is the decorated function 

384 return self.on(args[0].__name__)(args[0]) 

385 else: 

386 # the decorator was invoked with arguments 

387 def set_handler(handler): 

388 return self.on(handler.__name__, *args, **kwargs)(handler) 

389 

390 return set_handler

401 def emit(self, event, *args, **kwargs): 

402 """Emit a server generated SocketIO event. 

403 

404 This function emits a SocketIO event to one or more connected clients. 

405 A JSON blob can be attached to the event as payload. This function can 

406 be used outside of a SocketIO event context, so it is appropriate to 

407 use when the server is the originator of an event, outside of any 

408 client context, such as in a regular HTTP request handler or a 

409 background task. Example:: 

410 

411 @app.route('/ping') 

412 def ping(): 

413 socketio.emit('ping event', {'data': 42}, namespace='/chat') 

414 

415 :param event: The name of the user event to emit. 

416 :param args: A dictionary with the JSON data to send as payload. 

417 :param namespace: The namespace under which the message is to be sent. 

418 Defaults to the global namespace. 

419 :param to: Send the message to all the users in the given room, or to 

420 the user with the given session ID. If this parameter is not 

421 included, the event is sent to all connected users. 

422 :param include_self: ``True`` to include the sender when broadcasting 

423 or addressing a room, or ``False`` to send to 

424 everyone but the sender. 

425 :param skip_sid: The session id of a client to ignore when broadcasting 

426 or addressing a room. This is typically set to the 

427 originator of the message, so that everyone except 

428 that client receive the message. To skip multiple sids 

429 pass a list. 

430 :param callback: If given, this function will be called to acknowledge 

431 that the client has received the message. The 

432 arguments that will be passed to the function are 

433 those provided by the client. Callback functions can 

434 only be used when addressing an individual client. 

435 """ 

436 namespace = kwargs.pop('namespace', '/') 

437 to = kwargs.pop('to', None) or kwargs.pop('room', None) 

438 include_self = kwargs.pop('include_self', True) 

439 skip_sid = kwargs.pop('skip_sid', None) 

440 if not include_self and not skip_sid: 

441 skip_sid = flask.request.sid 

442 callback = kwargs.pop('callback', None) 

443 if callback: 

444 # wrap the callback so that it sets app app and request contexts 

445 sid = None 

446 original_callback = callback 

447 original_namespace = namespace 

448 if has_request_context(): 

449 sid = getattr(flask.request, 'sid', None) 

450 original_namespace = getattr(flask.request, 'namespace', None) 

451 

452 def _callback_wrapper(*args): 

453 return self._handle_event(original_callback, None, 

454 original_namespace, sid, *args) 

455 

456 if sid: 

457 # the callback wrapper above will install a request context 

458 # before invoking the original callback 

459 # we only use it if the emit was issued from a Socket.IO 

460 # populated request context (i.e. request.sid is defined) 

461 callback = _callback_wrapper 

462 self.server.emit(event, *args, namespace=namespace, to=to, 

463 skip_sid=skip_sid, callback=callback, **kwargs)

465 def call(self, event, *args, **kwargs): # pragma: no cover 

466 """Emit a SocketIO event and wait for the response. 

467 

468 This method issues an emit with a callback and waits for the callback 

469 to be invoked by the client before returning. If the callback isn’t 

470 invoked before the timeout, then a TimeoutError exception is raised. If 

471 the Socket.IO connection drops during the wait, this method still waits 

472 until the specified timeout. Example:: 

473 

474 def get_status(client, data): 

475 status = call('status', {'data': data}, to=client) 

476 

477 :param event: The name of the user event to emit. 

478 :param args: A dictionary with the JSON data to send as payload. 

479 :param namespace: The namespace under which the message is to be sent. 

480 Defaults to the global namespace. 

481 :param to: The session ID of the recipient client. 

482 :param timeout: The waiting timeout. If the timeout is reached before 

483 the client acknowledges the event, then a 

484 ``TimeoutError`` exception is raised. The default is 60 

485 seconds. 

486 :param ignore_queue: Only used when a message queue is configured. If 

487 set to ``True``, the event is emitted to the 

488 client directly, without going through the queue. 

489 This is more efficient, but only works when a 

490 single server process is used, or when there is a 

491 single addressee. It is recommended to always 

492 leave this parameter with its default value of 

493 ``False``. 

494 """ 

495 namespace = kwargs.pop('namespace', '/') 

496 to = kwargs.pop('to', None) or kwargs.pop('room', None) 

497 return self.server.call(event, *args, namespace=namespace, to=to, 

498 **kwargs)

500 def send(self, data, json=False, namespace=None, to=None, 

501 callback=None, include_self=True, skip_sid=None, **kwargs): 

502 """Send a server-generated SocketIO message. 

503 

504 This function sends a simple SocketIO message to one or more connected 

505 clients. The message can be a string or a JSON blob. This is a simpler 

506 version of ``emit()``, which should be preferred. This function can be

512 otherwise. 

513 :param namespace: The namespace under which the message is to be sent. 

514 Defaults to the global namespace. 

515 :param to: Send the message to all the users in the given room, or to 

516 the user with the given session ID. If this parameter is not 

517 included, the event is sent to all connected users. 

518 :param include_self: ``True`` to include the sender when broadcasting 

519 or addressing a room, or ``False`` to send to 

520 everyone but the sender. 

521 :param skip_sid: The session id of a client to ignore when broadcasting 

522 or addressing a room. This is typically set to the

524 that client receive the message. To skip multiple sids 

525 pass a list.

553 def run(self, app, host=None, port=None, **kwargs): # pragma: no cover 

554 """Run the SocketIO web server. 

555 

556 :param app: The Flask application instance. 

557 :param host: The hostname or IP address for the server to listen on. 

558 Defaults to 127.0.0.1. 

559 :param port: The port number for the server to listen on. Defaults to 

560 5000. 

561 :param debug: ``True`` to start the server in debug mode, ``False`` to 

562 start in normal mode. 

563 :param use_reloader: ``True`` to enable the Flask reloader, ``False`` 

564 to disable it. 

565 :param reloader_options: A dictionary with options that are passed to 

566 the Flask reloader, such as ``extra_files``, 

567 ``reloader_type``, etc. 

568 :param extra_files: A list of additional files that the Flask 

569 reloader should watch. Defaults to ``None``. 

570 Deprecated, use ``reloader_options`` instead. 

571 :param log_output: If ``True``, the server logs all incoming 

572 connections. If ``False`` logging is disabled. 

573 Defaults to ``True`` in debug mode, ``False`` 

574 in normal mode. Unused when the threading async 

575 mode is used. 

576 :param allow_unsafe_werkzeug: Set to ``True`` to allow the use of the 

577 Werkzeug web server in a production 

578 setting. Default is ``False``. Set to 

579 ``True`` at your own risk. 

580 :param kwargs: Additional web server options. The web server options 

581 are specific to the server used in each of the supported 

582 async modes. Note that options provided here will 

583 not be seen when using an external web server such 

584 as gunicorn, since this method is not called in that 

585 case. 

586 """ 

587 if host is None: 

588 host = '127.0.0.1' 

589 if port is None: 

590 server_name = app.config['SERVER_NAME'] 

591 if server_name and ':' in server_name: 

592 port = int(server_name.rsplit(':', 1)[1]) 

593 else: 

594 port = 5000

788 def _handle_event(self, handler, message, namespace, sid, *args): 

789 environ = self.server.get_environ(sid, namespace=namespace) 

790 if not environ: 

791 # we don't have record of this client, ignore this event 

792 return '', 400 

793 app = environ['flask.app'] 

794 with app.request_context(environ): 

795 if self.manage_session: 

796 # manage a separate session for this client's Socket.IO events 

797 # created as a copy of the regular user session 

798 if 'saved_session' not in environ: 

799 environ['saved_session'] = _ManagedSession(flask.session) 

800 session_obj = environ['saved_session'] 

801 if hasattr(flask, 'globals') and \ 

802 hasattr(flask.globals, 'request_ctx'): 

803 # update session for Flask >= 2.2 

804 ctx = flask.globals.request_ctx._get_current_object() 

805 else: # pragma: no cover 

806 # update session for Flask < 2.2 

807 ctx = flask._request_ctx_stack.top 

808 ctx.session = session_obj 

809 else: 

810 # let Flask handle the user session 

811 # for cookie based sessions, this effectively freezes the 

812 # session to its state at connection time 

813 # for server-side sessions, this allows HTTP and Socket.IO to 

814 # share the session, with both having read/write access to it 

815 session_obj = flask.session._get_current_object() 

816 flask.request.sid = sid 

817 flask.request.namespace = namespace 

818 flask.request.event = {'message': message, 'args': args} 

819 try: 

820 if message == 'connect': 

821 auth = args[1] if len(args) > 1 else None 

822 try: 

823 ret = handler(auth) 

824 except TypeError: 

825 ret = handler() 

826 else: 

827 ret = handler(*args) 

828 except ConnectionRefusedError: 

829 raise # let this error bubble up to python-socketio 

830 except: 

831 err_handler = self.exception_handlers.get( 

832 namespace, self.default_exception_handler) 

833 if err_handler is None: 

834 raise 

835 type, value, traceback = sys.exc_info() 

836 return err_handler(value) 

837 if not self.manage_session: 

838 # when Flask is managing the user session, it needs to save it 

839 if not hasattr(session_obj, 'modified') or \ 

840 session_obj.modified: 

841 resp = app.response_class() 

842 app.session_interface.save_session(app, session_obj, resp) 

843 return ret

846def emit(event, *args, **kwargs): 

847 """Emit a SocketIO event. 

848 

849 This function emits a SocketIO event to one or more connected clients. A 

850 JSON blob can be attached to the event as payload. This is a function that 

851 can only be called from a SocketIO event handler, as in obtains some 

852 information from the current client context. Example:: 

853 

854 @socketio.on('my event') 

855 def handle_my_custom_event(json): 

856 emit('my response', {'data': 42}) 

857 

858 :param event: The name of the user event to emit. 

859 :param args: A dictionary with the JSON data to send as payload. 

860 :param namespace: The namespace under which the message is to be sent. 

861 Defaults to the namespace used by the originating event. 

862 A ``'/'`` can be used to explicitly specify the global 

863 namespace. 

864 :param callback: Callback function to invoke with the client's 

865 acknowledgement. 

866 :param broadcast: ``True`` to send the message to all clients, or ``False`` 

867 to only reply to the sender of the originating event. 

868 :param to: Send the message to all the users in the given room, or to the 

869 user with the given session ID. If this argument is not set and 

870 ``broadcast`` is ``False``, then the message is sent only to the 

871 originating user. 

872 :param include_self: ``True`` to include the sender when broadcasting or 

873 addressing a room, or ``False`` to send to everyone 

874 but the sender. 

875 :param skip_sid: The session id of a client to ignore when broadcasting 

876 or addressing a room. This is typically set to the 

877 originator of the message, so that everyone except 

878 that client receive the message. To skip multiple sids 

879 pass a list. 

880 :param ignore_queue: Only used when a message queue is configured. If 

881 set to ``True``, the event is emitted to the 

882 clients directly, without going through the queue. 

883 This is more efficient, but only works when a 

884 single server process is used, or when there is a 

885 single addressee. It is recommended to always leave 

886 this parameter with its default value of ``False``. 

887 """ 

888 if 'namespace' in kwargs: 

889 namespace = kwargs['namespace'] 

890 else: 

891 namespace = flask.request.namespace 

892 callback = kwargs.get('callback') 

893 broadcast = kwargs.get('broadcast') 

894 to = kwargs.pop('to', None) or kwargs.pop('room', None) 

895 if to is None and not broadcast: 

896 to = flask.request.sid 

897 include_self = kwargs.get('include_self', True) 

898 skip_sid = kwargs.get('skip_sid') 

899 ignore_queue = kwargs.get('ignore_queue', False) 

900 

901 socketio = flask.current_app.extensions['socketio'] 

902 return socketio.emit(event, *args, namespace=namespace, to=to, 

903 include_self=include_self, skip_sid=skip_sid, 

904 callback=callback, ignore_queue=ignore_queue)

907def call(event, *args, **kwargs): # pragma: no cover 

908 """Emit a SocketIO event and wait for the response. 

909 

910 This function issues an emit with a callback and waits for the callback to 

911 be invoked by the client before returning. If the callback isn’t invoked 

912 before the timeout, then a TimeoutError exception is raised. If the 

913 Socket.IO connection drops during the wait, this method still waits until 

914 the specified timeout. Example:: 

915 

916 def get_status(client, data): 

917 status = call('status', {'data': data}, to=client) 

918 

919 :param event: The name of the user event to emit. 

920 :param args: A dictionary with the JSON data to send as payload. 

921 :param namespace: The namespace under which the message is to be sent. 

922 Defaults to the namespace used by the originating event. 

923 A ``'/'`` can be used to explicitly specify the global 

924 namespace. 

925 :param to: The session ID of the recipient client. If this argument is not 

926 given, the event is sent to the originating client. 

927 :param timeout: The waiting timeout. If the timeout is reached before the 

928 client acknowledges the event, then a ``TimeoutError`` 

929 exception is raised. The default is 60 seconds. 

930 :param ignore_queue: Only used when a message queue is configured. If 

931 set to ``True``, the event is emitted to the 

932 client directly, without going through the queue. 

933 This is more efficient, but only works when a 

934 single server process is used, or when there is a 

935 single addressee. It is recommended to always leave 

936 this parameter with its default value of ``False``. 

937 """ 

938 if 'namespace' in kwargs: 

939 namespace = kwargs['namespace'] 

940 else: 

941 namespace = flask.request.namespace 

942 to = kwargs.pop('to', None) or kwargs.pop('room', None) 

943 if to is None: 

944 to = flask.request.sid 

945 timeout = kwargs.get('timeout', 60) 

946 ignore_queue = kwargs.get('ignore_queue', False) 

947 

948 socketio = flask.current_app.extensions['socketio'] 

949 return socketio.call(event, *args, namespace=namespace, to=to, 

950 ignore_queue=ignore_queue, timeout=timeout)

953def send(message, **kwargs): 

954 """Send a SocketIO message. 

955 

956 This function sends a simple SocketIO message to one or more connected 

957 clients. The message can be a string or a JSON blob. This is a simpler 

958 version of ``emit()``, which should be preferred. This is a function that 

959 can only be called from a SocketIO event handler. 

960 

961 :param message: The message to send, either a string or a JSON blob. 

962 :param json: ``True`` if ``message`` is a JSON blob, ``False`` 

963 otherwise. 

964 :param namespace: The namespace under which the message is to be sent. 

965 Defaults to the namespace used by the originating event. 

966 An empty string can be used to use the global namespace. 

967 :param callback: Callback function to invoke with the client's 

968 acknowledgement. 

969 :param broadcast: ``True`` to send the message to all connected clients, or 

970 ``False`` to only reply to the sender of the originating 

971 event. 

972 :param to: Send the message to all the users in the given room, or to the 

973 user with the given session ID. If this argument is not set and 

974 ``broadcast`` is ``False``, then the message is sent only to the 

975 originating user. 

976 :param include_self: ``True`` to include the sender when broadcasting or 

977 addressing a room, or ``False`` to send to everyone 

978 but the sender. 

979 :param skip_sid: The session id of a client to ignore when broadcasting 

980 or addressing a room. This is typically set to the 

981 originator of the message, so that everyone except 

982 that client receive the message. To skip multiple sids 

983 pass a list. 

984 :param ignore_queue: Only used when a message queue is configured. If 

985 set to ``True``, the event is emitted to the 

986 clients directly, without going through the queue. 

987 This is more efficient, but only works when a 

988 single server process is used, or when there is a 

989 single addressee. It is recommended to always leave 

990 this parameter with its default value of ``False``. 

991 """ 

992 json = kwargs.get('json', False) 

993 if 'namespace' in kwargs: 

994 namespace = kwargs['namespace'] 

995 else: 

996 namespace = flask.request.namespace 

997 callback = kwargs.get('callback') 

998 broadcast = kwargs.get('broadcast') 

999 to = kwargs.pop('to', None) or kwargs.pop('room', None) 

1000 if to is None and not broadcast: 

1001 to = flask.request.sid 

1002 include_self = kwargs.get('include_self', True) 

1003 skip_sid = kwargs.get('skip_sid') 

1004 ignore_queue = kwargs.get('ignore_queue', False) 

1005 

1006 socketio = flask.current_app.extensions['socketio'] 

1007 return socketio.send(message, json=json, namespace=namespace, to=to, 

1008 include_self=include_self, skip_sid=skip_sid, 

1009 callback=callback, ignore_queue=ignore_queue)
